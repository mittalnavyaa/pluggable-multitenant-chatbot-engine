How This Integration Works

1. Widget Embed

A single <script> tag loads the widget bundle. A single <envoy-chatbot> HTML tag places the widget. No framework required.

2. Branding Load

On mount, the widget fetches branding from GET /api/v1/products/:botId and applies colors, fonts, and content dynamically.

3. Chat Request

User messages are sent to POST /api/v1/chat on your host server. The Backend SDK middleware validates, signs, and forwards the request to the core engine.

4. Streaming Response

The bot response streams back as Server-Sent Events. The widget renders each token as it arrives, giving a real-time typing effect.
