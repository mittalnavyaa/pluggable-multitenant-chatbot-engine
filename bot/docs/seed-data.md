# Seed Data

## Overview

The seed data models seven internal products connected to the centralized chatbot platform:

| Product ID | Product Name | Primary Use |
| --- | --- | --- |
| `tensor` | Tensor | Analytics and reporting assistance |
| `admissions` | Admissions | Applicant and admissions process support |
| `internal-support` | Internal Support | IT and workplace support |
| `hr-portal` | HR Portal | HR policy and employee support |
| `placement-cell` | Placement Cell | Campus placement operations |
| `website-analyzer` | Website Analyzer | SEO, crawl, and website audit assistance |
| `knowledge-base` | Knowledge Base | Enterprise knowledge discovery |

Seed files are stored in [seed/products.json](../seed/products.json), [seed/themes.json](../seed/themes.json), [seed/sample_documents.json](../seed/sample_documents.json), and [seed/sample_payloads.json](../seed/sample_payloads.json).

## Product Records

Each product includes:

| Field | Description |
| --- | --- |
| `product_id` | Stable internal product key |
| `product_name` | Human-readable name |
| `service_token_hash` | Hash of the internal service token; marked `TODO - Requires business confirmation` until the business-approved token rotation process provides values |
| `is_active` | Whether the product can use chatbot APIs |
| `branding_config` | Widget theme and display configuration |

Example:

```json
{
  "product_id": "tensor",
  "product_name": "Tensor",
  "service_token_hash": "TODO - Requires business confirmation: Tensor internal service token hash",
  "is_active": true,
  "branding_config": {
    "primaryColor": "#2563EB",
    "accentColor": "#14B8A6",
    "widgetTitle": "Tensor Assistant",
    "welcomeMessage": "Ask Tensor Assistant about analytics, models, and reports."
  }
}
```

## Sample Documents

Documents represent realistic internal knowledge sources. Each document belongs to exactly one product.

```json
{
  "document_id": "doc_hr_leave_policy",
  "product_id": "hr-portal",
  "title": "Leave and Attendance Policy",
  "source": "hr-portal/policies/leave-and-attendance.md",
  "language": "en",
  "document_type": "policy",
  "metadata": {
    "owner": "People Operations",
    "classification": "internal"
  }
}
```

## Sample Payloads

Qdrant payload seed records include product-specific chunk metadata:

```json
{
  "product_id": "placement-cell",
  "document_id": "doc_placement_interview_process",
  "source": "placement-cell/process/interview-process.md",
  "title": "Interview Process Guide",
  "language": "en",
  "created_at": "2026-06-29T09:00:00Z",
  "chunk_index": 0,
  "document_type": "guide",
  "classification": "internal",
  "owner": "Career Services"
}
```

## Usage Notes

1. Replace each `TODO - Requires business confirmation` service token hash with a business-approved hash generated by the backend token hashing utility.
2. Load `themes.json` into product configuration tooling or use it to populate `branding_config`.
3. Use `sample_documents.json` to test document upload and ingestion.
4. Use `sample_payloads.json` to validate Qdrant filtering and citation behavior.

Plaintext service tokens are intentionally not stored in seed data. Token values must come from the approved internal secrets process.
