# API Payload & Interaction Specifications

This document serves as the authoritative, versioned specification describing the communication protocols shared between the Frontend Widget (`@envoy-ai/chatbot-ui`), Backend SDK (`@envoy-ai/chatbot-backend-sdk`), and the Core FastAPI RAG Engine.

---

## 1. Global Interaction Flow

```
Widget (<envoy-chatbot>)
      │
      │ 1. GET /api/v1/widget/init?bot_id={id}
      │ 2. POST /api/v1/chat (Prompt JSON)
      ▼
Backend SDK Middleware (Host server)
      │
      │ 3. POST /api/v1/chat/stream
      │    - Inject X-Envoy-Signature (HMAC-SHA256)
      │    - Inject X-Envoy-API-Key
      ▼
Core FastAPI Backend (Central Hub)
```

---

## 2. API Versioning Policy

- All platform endpoints are version-prefixed (e.g. `/api/v1/...`).
- Minor, non-breaking modifications (such as optional response metadata fields) will not increment the versioning route path.
- Breaking alterations will spawn subsequent major version endpoints (e.g. `/api/v2/...`).

---

## 3. Required HTTP Headers

| Header | Required | Purpose / Validation |
| :--- | :--- | :--- |
| `Content-Type` | Yes | Must be `application/json` (except SSE streams). |
| `X-Envoy-API-Key` | Yes | Private platform credentials injected by the server-side SDK. |
| `X-Envoy-Product-ID` | Yes | Identifies target tenant product ID workspace. |
| `X-Envoy-Signature` | Yes | HMAC-SHA256 hash verifying request payload authenticity. |
| `X-Envoy-Timestamp` | Yes | Unix epoch timestamp used in HMAC. Must be within 5 mins of server local time. |
| `X-Envoy-Nonce` | Yes | 16-byte random hex string preventing request replay attacks. |
| `X-Correlation-ID` | No | Trace ID generated by host systems to log queries lifecycle. |

## 4. Endpoints Catalog & Payload Schemas

### 1. Widget Initialization
- **Route**: `GET /api/v1/widget/init`
- **Goal**: Returns layouts, visual themes, assets, and toggled configuration feature flags.
- **Request Parameters**:
  - `bot_id` (Query string, required).

### 2. Send Chat Message
- **Route**: `POST /api/v1/chat`
- **Goal**: Dispatches query prompt block. Supports either final JSON message or text chunk streams.
- **Request Body**:
  - `bot_id`: UUID/string, required.
  - `conversation_id`: UUID/string, required.
  - `prompt`: String, required, maximum length 4000 characters.
  - `stream`: Boolean, required.
  - `metadata`: Object, optional.
- **JSON Response Format** (if `stream: false`):
```json
{
  "success": true,
  "message": {
    "id": "msg_8f8a12bc",
    "conversation_id": "conv_99018",
    "sender": "bot",
    "text": "The SDK is configured by passing your private API credentials and tenant identifiers.",
    "timestamp": "2026-07-09T10:45:00Z"
  }
}
```

---

### 3. Server-Sent Events (SSE) Streaming
- **Route**: `POST /api/v1/chat/stream` (Core internal gateway proxy endpoint).
- **Format**: `text/event-stream` chunks sequence:
  - **Token text chunk**:
    ```
    data: {"event": "text", "text": "Hello"}
    
    ```
  - **Completion chunk**:
    ```
    data: {"event": "done", "message_id": "msg_90a12"}
    
    ```
  - **Exception failure chunk**:
    ```
    data: {"event": "error", "error": "LLM Timeout occurred"}
    
    ```

### 4. Health Check
- **Route**: `GET /health`
- **Response JSON Payload**:
```json
{
  "status": "healthy",
  "timestamp": "2026-07-09T10:45:00Z",
  "version": "1.0.0"
}
```

### 5. Unified API Error Schema
All services must reject malformed requests or server faults using this single structure:
```json
{
  "success": false,
  "error": {
    "code": "BAD_REQUEST",
    "message": "Parameter 'prompt' exceeds the maximum permitted length of 4000 characters",
    "details": {
      "parameter": "prompt",
      "limit": 4000,
      "received": 4001
    },
    "correlation_id": "trace-91b3-abc",
    "timestamp": "2026-07-09T10:45:00Z",
    "retryable": false
  }
}
```
Possible Error Codes:
- `BAD_REQUEST`: Payload parameter missing or invalid formatting.
- `UNAUTHORIZED`: Request signature header missing or invalid.
- `FORBIDDEN`: Client is not authorized to query target bot.
- `TIMEOUT`: Gateway timeout occurred connecting to agent services.
- `SERVICE_UNAVAILABLE`: Backend core engines are unreachable.
